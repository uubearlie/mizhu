# 商品状态流转

## 需求

管理商品从录入到返现完成的完整生命周期，支持状态切换和异常恢复。

## 规格说明

### 状态定义

| 状态 | 说明 |
|------|------|
| 待收货 | 默认状态，返现条目 dueDate 未计算，倒计时不启动 |
| 待返现 | 已确认收货，返现条目计算 dueDate，启动14天倒计时 |
| 已完成 | 所有返现已收齐（待返归零自动切换） |
| 已取消 | 软删除，可恢复 |

注意：状态是 Item 级别的概念，CashbackEntry 上不存储收款状态。收款追踪通过 PaymentRecord 实现。

### 状态切换动作

**确认收货：**
- 触发：单个确认 或 批量确认
- 效果：待收货 → 待返现
- 设置确认收货日期（默认今天，最晚不超过当天）
- 为所有 CashbackEntry 计算 dueDate（大套购 dueDate=null 除外）
- 平台保价类型：dueDate = 收货日期
- 触发动态重算（品牌套购/迷住保价）

**收到返现：**
- 触发：收到返现弹窗操作
- 新增收款记录（金额 + 付款方 + 日期）
- 更新累计已返金额
- **校验：本次收款金额不允许超过对应付款方的待返金额**
  - 店铺收款金额不得超过店铺待返
  - 迷住收款金额不得超过迷住待返
  - 超过时阻止保存，提示具体可收金额
- 待返归零 → 自动切换为已完成
- 已完成状态下仍可继续收款（追加收款）
- 收款记录可删除，删除后重新计算待返，如待返 > 0 则状态回退到待返现

**取消订单：**
- 前置条件：无收款记录 且 未参与大套购结算
- 效果：切换为已取消
- 已取消的订单不参与大套购计算
- 已取消的订单不出现在待返清单中
- 已取消的订单不显示保价入口

**重置状态：**
- 效果：回到待收货
- 删除所有 PaymentRecord（收款记录）
- 清空 confirmDate
- 所有 CashbackEntry 的 dueDate 清空（倒计时重置）
- 用于修正错误信息（如退换货后重新等待物流）

**删除：**
- 硬删除，连级删除所有关联数据（返现条目、收款记录）
- 不受当前状态限制

### 保价时限

- 保价可用期：下单日期 → 确认收货日期 + 100天
- 超过100天后，保价按钮变为不可点击（灰色）
- 已取消状态：保价按钮不显示

### 快速保价弹窗

从商品清单或商品详情页的"保价"按钮触发，包含三个区域：

**平台保价区域：**
- 展示历史保价记录列表（日期 + 金额），每条可删除
- 显示累计金额
- 底部输入框 + "添加"按钮，点击立即在列表中新增一条（不写后端）

**店铺保价区域：**
- 结构同平台保价，独立累计

**迷住保价区域：**
- 保价类型选择（一类/二类）
- 好价输入框

**弹窗底部操作：**
- "取消"：关闭弹窗，不保存新增记录
- "确认保价"：关闭弹窗，将弹窗内所有新增的保价记录写入后端
