# 返现计算引擎

## 需求

自动计算所有返现类型、保价差价、大套购分摊，实时聚合到手价和待返金额。

## 规格说明

### 返现类型映射（固定映射，选择类型自动带出付款方和到账规则）

| 类型 | 付款方 | 到账规则 |
|------|--------|---------|
| 平台保价退款 | 店铺 | 收货后即时 |
| 店铺保价退款 | 店铺 | 收货+14天 |
| 下单即返 | 店铺 | 收货+14天 |
| 小红书晒单 | 店铺 | 收货+14天 |
| 组合购 | 店铺 | 收货+14天 |
| 其他(自定义) | 店铺 | 收货+14天 |
| 品牌套购 | 店铺 | 收货+14天 |
| 迷住返(加返) | 迷住 | 收货+14天 |
| 舰长神券 | 迷住 | 收货+14天 |
| 装修神券 | 迷住 | 收货+14天 |
| 迷住保价 | 迷住 | 收货+14天 |
| 大套购分摊 | 迷住 | 手动结算 |

### 品牌套购计算

```
基准 = 下单价 - 平台保价退款 - 店铺保价退款 - Σ(店铺侧手填条目)
返现 = 基准 × (1 - 系数)
```

- 基准 ≤ 0 时 → 归零，不产生品牌套购返现
- 系数由用户填入（如 0.98），填了立即生成 CashbackEntry(type='brand')
- 基准依赖其他 Entry 金额，每次 Item 更新后重算
- "店铺侧手填条目" = instant_cashback + xhs_post + combo + other 的金额之和

### 大套购计算（活动级别）

**预估阶段（商品创建时自动生成）：**
- 预估分摊 = price × 0.05（固定常量，写死不存储）
- 自动生成 CashbackEntry { type:'big_purchase', amount:预估值, isEstimated:true, dueDate:null }
- 预估完全独立，不依赖其他订单

**分档拆分算法（档位不共享）：**
- 金额按从高到低的档位依次消耗
- 每个档位计算：该档位返现 = floor(剩余金额 / 档位阈值) × 档位返现
- 剩余金额 = 剩余金额 mod 档位阈值

**分摊算法：**
- 按各商品实付金额占本批总额的比例分摊

**结算规则：**
- 支持分批结算：用户选择部分订单，独立计算分档
- 支持全部结算：选中所有未结算订单
- 已结算的订单不出现在结算选择列表中
- 撤销只支持整批撤销，不支持单件
- 结算时 Entry.amount 更新为实际分摊值，isEstimated → false
- 结算时自动生成 PaymentRecord(payer='迷住')
- 撤销时 Entry.amount 回退为 price × 0.05，isEstimated → true，对应 PaymentRecord 删除
- 恢复已取消订单 → 累入大套购未结算金额，已结算不受影响

### 迷住保价计算

用户输入：迷住保价到手价 + 类型（一类/二类）
迷住保价保的是到手价，直接与原到手价比差，不需要重算扣减项。

**一类保价：**
```
原一类到手价 = 原下单价 - 单品返现 - 平台保价 - 店铺保价
保价返现 = 原一类到手价 - 迷住保价到手价
```

**二类保价：**
```
原二类到手价 = 原下单价 - 单品返现 - 店铺活动 - 品牌套购
              - 大套购 - 平台保价 - 店铺保价
保价返现 = 原二类到手价 - 迷住保价到手价
```

**保价返现规则：**
- 保价返现 > 0 → 生成迷住保价返现条目
- 保价返现 ≤ 0 → 记录迷住保价到手价但不生成返现条目（保价到手价不优于原到手价）

### 平台/店铺保价（多次累计）

- 平台保价和店铺保价支持多次添加，每次新增一笔返现条目
- 每笔保价记录包含：日期、金额、类型（平台/店铺）
- 保价弹窗中展示历史记录列表（含累计金额），每条记录可删除
- 新增保价记录时立即在弹窗中显示，点击"确认保价"后统一写入后端
- 所有保价金额参与品牌套购基准计算

### 到账日期推算

- 即时类型（平台保价）：应到日期 = 确认收货日期
- +14天类型：应到日期 = 确认收货日期 + 14天
- 手动结算（大套购）：无应到日期
- 未确认收货：所有返现条目状态为"未激活"，倒计时不启动

### 聚合计算（实时）

- 店铺侧合计 = Σ(Entry where payer='店铺')
- 迷住侧合计 = Σ(Entry where payer='迷住')（自动包含大套购预估/实际）
- 到手价 = price - 店铺侧合计 - 迷住侧合计
- 已返合计 = Σ(PaymentRecord)
- 店铺待返 = 店铺侧合计 - Σ(PaymentRecord where payer='店铺')
- 迷住待返 = 迷住侧合计 - Σ(PaymentRecord where payer='迷住')

### 动态重算（每次 Item 更新后触发）

Item 信息更新时，自动重算以下 Entry：

1. 品牌套购 Entry(type='brand')：基于最新的店铺侧手填条目重算基准和返现
2. 迷住保价 Entry(type='mizhu_protect')：基于最新的相关 Entry 重算差价
3. 大套购 Entry(type='big_purchase')：仅 isEstimated=true 时重算为 price×0.05，已固定跳过

### 金额精度

- 所有金额计算使用整数分（×100）
- 显示时保留 2 位小数
- 大套购预估比例 0.05 为固定常量
